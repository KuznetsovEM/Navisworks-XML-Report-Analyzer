ПРОЕКТ: Clash Matrix Analyzer для Navisworks

=== ОБЩЕЕ ОПИСАНИЕ ===
Это standalone веб-приложение (один HTML-файл) для удобного анализа отчётов коллизий (clash reports), экспортированных из Autodesk Navisworks в формате XML.

Основная задача — представить коллизии в виде удобной матрицы с иерархической группировкой по свойствам Элемента 1 и Элемента 2, с мощными фильтрами и возможностью быстрого поиска проблем.

Приложение полностью работает в браузере, без установки и без сервера.

=== ОСНОВНЫЕ ВОЗМОЖНОСТИ ===
- Загрузка XML-отчётов Navisworks (Проверка 1.xml и т.п.)
- Иерархическая группировка строк (по свойствам Элемента 1)
- Иерархическая группировка столбцов (по свойствам Элемента 2)
- Перетаскивание уровней иерархии (drag & drop) для изменения порядка
- Многоуровневая шапка таблицы с объединением ячеек
- Фильтры:
  * По проверке (Clashtest name)
  * Глобальный поиск по всем параметрам
  * Дополнительные фильтры по любым свойствам с операторами:
    - содержит
    - не содержит
    - равно
    - не равно
    - начинается с
  * Выбор стороны применения фильтра (Оба элемента / Только Элемент 1 / Только Элемент 2)
- Фиксированная шапка и левые колонки при прокрутке
- Подсчёт количества коллизий на пересечениях
- Экспорт в CSV
- Поддержка динамических свойств из XML (objectattribute + smarttags)

=== КАК ЗАПУСТИТЬ ===
1. Открой файл index.html в любом современном браузере (Chrome, Edge, Firefox).
2. Нажми кнопку «Выбрать файл» и загрузи нужный .xml файл из папки «Отчет Navisworks».
3. Настрой иерархию строк и столбцов.
4. При необходимости добавляй фильтры.

Приложение не требует интернета после загрузки (кроме CDN Tailwind при первом открытии).

=== ФОРМАТ ВХОДНЫХ ДАННЫХ ===
Приложение ожидает XML в формате экспорта Navisworks Clash Report.

Ключевые элементы:
- <exchange>
  - <batchtest>
    - <clashtests>
      - <clashtest name="Название проверки">
        - <clashresults>
          - <clashresult ...>
            - <clashobjects>
              - <clashobject> (Элемент 1)
                - <objectattribute> (много)
                - <layer>
                - <pathlink><node>...</node></pathlink>
                - <smarttags>
              - <clashobject> (Элемент 2)

Свойства элементов собираются из:
- objectattribute (name + value)
- layer
- pathlink (разбирается на Модель, Категория и т.д.)
- smarttags

Каждому clashresult присваивается testName = имя clashtest.

=== КЛЮЧЕВЫЕ ПОНЯТИЯ ===
- Элемент 1 (item1) — первый объект в коллизии
- Элемент 2 (item2) — второй объект в коллизии
- Иерархия строк — группировка по свойствам Элемента 1
- Иерархия столбцов — группировка по свойствам Элемента 2
- testName / Проверка — название теста (Clashtest name)

=== РАБОТА С ИЕРАРХИЕЙ ===
- Добавляй уровни через выпадающий список + кнопку "+ уровень"
- Перетаскивай чипы мышкой, чтобы менять порядок уровней
- Можно использовать свойство "Проверка" (testName) как один из уровней

=== РАБОТА С ФИЛЬТРАМИ ===
1. Фильтр по проверке — чекбоксы
2. Глобальный поиск — ищет по всем текстовым значениям коллизии
3. Дополнительные фильтры:
   - Выбери свойство
   - Выбери сторону (Оба / Только Элемент 1 / Только Элемент 2)
   - Выбери оператор (включая "не содержит")
   - Введи значение

Все фильтры работают совместно (AND).

=== ТЕХНИЧЕСКАЯ ИНФОРМАЦИЯ ===
- Полностью клиентское приложение (HTML + Vanilla JS)
- Использует Tailwind CSS через CDN
- Парсинг XML через DOMParser
- Все данные хранятся в памяти браузера после загрузки файла
- Нет сохранения состояния между перезагрузками страницы

=== ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ ===
- Работает только с XML-экспортом Navisworks (не с HTML-отчётами)
- При очень большом количестве коллизий (>10-15k) может тормозить браузер
- Свойства с одинаковыми именами в objectattribute и smarttags могут перезаписываться
- Экспорт CSV — упрощённый (только текущая матрица)
